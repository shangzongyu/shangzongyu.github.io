# AI Agents Configuration

本项目的 AI 辅助配置，定义了各 AI Agent 的行为和职责边界。

## 项目概述

- **类型**: Hugo 个人博客
- **主题**: hugo-theme-stack v3.33.0
- **内容语言**: 中文
- **主要编辑器**: Obsidian / Typora
- **图片托管**: GitHub (upgit/piclist)

## 核心原则

1. **中文优先**: 所有文章和文档应使用中文
2. **Markdown 规范**: 遵循 Hugo 博客的 Markdown 格式
3. **简洁实用**: 内容应简洁、实用、有价值
4. **技术准确**: 技术性内容需要准确验证

## Agent 行为指南

### 写作和内容创建

- **语言**: 必须使用中文
- **格式**: 使用 Markdown，遵循 Hugo frontmatter 规范
- **结构**: 建议包含标题、摘要、正文
- **链接**: 使用相对路径，便于 GitHub Pages 部署
- **图片**: 使用 GitHub 作为图床，确保链接可访问

### 代码示例

- **语言**: 根据内容选择合适的编程语言
- **注释**: 代码应有必要的中文注释
- **可运行**: 提供的代码示例应可运行或有明确说明
- **格式化**: 遵循各语言的代码风格规范

### 技术验证

- **命令行示例**: 验证命令的正确性
- **配置文件**: 确保配置文件格式正确
- **安装步骤**: 验证安装步骤的可行性
- **依赖说明**: 明确说明依赖和版本要求

### 文档引用

- **官方文档**: 优先引用官方文档
- **时效性**: 引用最新的文档和版本
- **中文文档**: 优先使用中文文档，如无则使用英文文档
- **链接稳定性**: 确保引用的链接可访问

## 禁止行为

- ❌ 不要生成英文内容（除非是代码或命令）
- ❌ 不要创建不必要的文件
- ❌ 不要修改配置文件（除非明确要求）
- ❌ 不要创建 PR 或提交代码（除非明确要求）
- ❌ 不要运行构建或部署命令（除非明确要求）

## 文件组织

### 文章位置

- 博客文章: `content/post/`
- 页面: `content/page/`
- 分类: `content/categories/`

### 命名规范

- 文章: `YYYY-标题.md` 或 `YYYY-类别_标题.md`
- 使用连字符 `-` 或下划线 `_` 分隔
- 避免使用特殊字符

### Frontmatter 模板

```yaml
---
title: "文章标题"
date: 2026-04-18T00:00:00+08:00
draft: false
categories: ["工具推荐"]
tags: ["工具", "软件"]
---
```

## 常用命令

### 本地开发

```bash
# 启动本地服务器
hugo server -D

# 构建静态网站
hugo

# 构建并部署
./start.sh
```

### 内容管理

- 在 `content/post/` 下创建新文章
- 使用 Obsidian 或 Typora 编辑
- 提交到 GitHub 自动部署

## 与 AI 交互

### 提问方式

- 清晰描述需求
- 指明文件位置（如有）
- 说明期望的输出格式
- 提供上下文（如相关文章或主题）

### 期望输出

- 中文内容
- Markdown 格式
- 代码高亮
- 必要的说明和注释
